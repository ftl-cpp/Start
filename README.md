# Регламент
## Или как работаем

1. Переходим по ссылке, которую вы получили/получите
2. Соглашаемся на создание репозитория
3. Клонируем к себе репозиторий
> [!TIP]
> На Windows
> 
> Открываем cmd
> 
> Переходим в нужную директорию ```cd ПУТЬ```
> 
> Пишем ```git clone URL_НА_ВАШ_РЕПОЗИТОРИЙ```
4. Открываем папку как проект в VS Code
> [!TIP]
> Можно написать в консоли из папки проекта ```code .```
5. Создаём ветку с текущим дедлайном и переключаемся на неё
> [!TIP]
> Создание ветки ```git branch DEADLINE_НОМЕР```
> 
> Переключение на ветку ```git checkout DEADLINE_НОМЕР```
> 
> Проверить текущую ветку ```git branch```
6. Читаем README.md и выполнем задачу
7. Можно (буду рад видеть!) создавать коммиты во время работы
> [!TIP]
> Для добавления файлов ```git add ПУТЬ_К_ФАЙЛУ.РАСШИРЕНИЕ```
> 
> Для коммита после добавления всех желаемых файлов ```git commit -m "НАЗВАНИЕ КОММИТА ПО СЕМАНТИКЕ"```

> [!WARNING]
> **Добавляем ТОЛЬКО файлы `.cpp`, `.h`, `.hpp`, `CMakeLists.txt`**
>
> Никаких файлов компиляции, сторонних библиотек и бинарников в репозитории!
8. Чтобы сдать работу убеждаемся, что все файлы в нужной ветке и закомичены
9. Пушим на удалённый репозиторий
> [!TIP]
> ```git push```
10. Переходим в Github и создаём Pull Request из ветки ``DEADLINE_НОМЕР`` в ``main``. **Не нажимаем Merge ни в каком виде**
11. **Время сдачи работы - время создания <ins>Pull Request</ins>**
12. Ждём Code Review (уведомления приходят на почту от GitHub, но можно и просто проверять Pull Request)
13. Приходим на сдачу и защищаемся (если это предусмотрено)

> [!IMPORTANT]
> Задаём смело вопросы, если они есть, в любой момент!

# Да кто этот ваш CMakeLists и как с ним жить!
→ [Маленькая статья про него + ссылка на установку в начале](https://habr.com/ru/articles/904992/)

→ [Менее наглядная, но качественная статья](https://neerc.ifmo.ru/wiki/index.php?title=CMake_Tutorial)

Тут оставлю простые шаблоны для CMakeLists.txt

Главный CMakeLists - в корне проекта
```
cmake_minimum_required(VERSION 3.12)     - Минимальная версия, которую мы требуем от сборщика

project(
    PROJECT_NAME      - Тут любое название проекта, вероятнее всего будет установлено до вас
    LANGUAGES CXX
)

set(CMAKE_CXX_STANDARD 23)     - Стандарт языка CPP. Можно любой кроме 26. Советую 23

include_directories(lib)       - Говорим сборщику, зайди в эту папку и поищи ещё CMakeLists.txt там для библиотек
ИЛИ ИЛИ ИЛИ
add_subdirectory(lib)          - Говорим сборщику, зайди в эту папку и поищи ещё CMakeLists.txt какие-то (например, если есть дробление внутри на папки)

add_subdirectory(bin)          - Говорим сборщику, зайди в эту папку и поищи ещё CMakeLists.txt какие-то
```

CMakeLists в папке lib
```
add_library(любое_название_для_сборщика MyLibrary.cpp)  - Если у нас в lib лежат сами исходники
add_library(любое_название_для_сборщика_2 MySecondTheBestLibrary.cpp)  - Если у нас в lib лежат сами исходники

add_subdirectory(название_папки_228) - если в lib есть дробление на другие папки
```

CMakeLists в папке `название_папки_228`
```
add_library(название_папки_228 MyLibrary.cpp)

target_include_directories(название_папки_228 PUBLIC ${PROJECT_SOURCE_DIR}/lib) - подвязываемся к lib
```

CMakeLists в bin
```
add_executable(название_проекта main.cpp)

target_link_libraries(название_проекта PUBLIC перечисляем_через_пробел_название_модулей_из_add_library_что_здесь_нужны)
```

## А как сие чудо запустить?

В консоли из корня проекта прописываем следующее (Windows)
```
mkdir build - создаём директорию build
cd build - переходим в неё
cmake .. - инициализируем cmake
cmake --build . - заставляем сбилдиться сюда, где находимся (т.е. в build)
```
Если ошибок нет, то в папке Debug будет `PROJECT_NAME.exe`. Папка `Debug` может находится в разных местах, в зависимости от нахождения `main.cpp`. Если `main.cpp` в `bin`, то папка `Debug` будет в `build\bin`
